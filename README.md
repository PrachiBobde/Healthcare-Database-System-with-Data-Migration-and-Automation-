## Overview:

This project focuses on designing and implementing a robust relational database system for a hospital by migrating data from an Excel-based system. The solution improves data integrity, scalability, and operational efficiency while enforcing critical business rules.



## Problem Statement:

The hospital relied on Excel files to manage patient records, appointments, billing, and reports. As operations scaled, this led to:

* Data inconsistencies and duplication.

* Lack of relationships between entities.

* Invalid and ambiguous entries.

* No control over scheduling conflicts.

* Unrestricted access to sensitive data.

* Inability to generate structured reports.

 
## Solution:

Developed a normalized SQL database with enforced constraints, relationships, and automation to:

* Ensure data integrity and consistency.
  
* Establish proper entity relationships.

* Automate appointment validation rules.
  
* Implement role-based access control.
  
* Enable department-wise revenue reporting.

## Data Migration:


* Cleaned and transformed Excel data.

* Removed duplicates and inconsistencies.

* Mapped raw data into structured relational tables.

* Ensured seamless migration with validation checks.<br>

## Tech Stack:

* SQL 

* Excel (Data Source)

## Impact:

* Reduced data redundancy and errors.

* Improved query performance and reporting.

* Enforced strict data governance.

* Enabled scalable and maintainable system design
